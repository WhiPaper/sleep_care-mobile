# AGENTS.md

이 저장소에서 작업하는 에이전트는 코드를 고칠 때마다 **읽기 쉬운 한글 주석**과 **현실적인 문서화**를 함께 챙긴다. 주석은 코드를 장식하는 문장이 아니라, 다음 개발자가 흐름을 빨리 이해하게 하는 안전장치다.

## 프로젝트 구조

- `app`: Android 모바일 앱. Compose UI, Hilt, Room/DataStore, Health Connect, Wear OS Data Layer, Raspberry Pi WSS/NSD 연동을 담당한다.
- `watch`: Wear OS 앱. 휴대폰 명령을 받아 워치 세션을 시작하고 심박 샘플과 알림을 처리한다.
- `watch-contracts`: 모바일 앱과 워치 앱이 공유하는 메시지 경로, 모델, JSON 코덱이다.
- `docs`: 기획, 프로토콜, 데이터, 운영 문서가 들어간다.
- `stitch_exports`: 디자인 산출물이다. 구현 참고용으로 보고, 불필요한 수정은 피한다.

## 주석 작성 원칙

- 주석은 기본적으로 **한글**로 작성한다.
- "무엇을 하는지"보다 **왜 필요한지**, **어떤 흐름에서 호출되는지**, **주의할 조건이 무엇인지**를 설명한다.
- 함수명과 코드만 봐도 명확한 내용은 주석으로 반복하지 않는다.
- 복잡한 흐름에는 블록 시작 지점에 짧은 설명을 둔다.
- 도메인 모델, Repository, DataSource, 서비스, 프로토콜 코덱처럼 경계가 되는 파일에는 파일/클래스 단위 주석을 우선 작성한다.
- UI 파일에서는 화면의 역할, ViewModel이 합치는 데이터, 상태 변환 함수를 설명한다.
- 임시 구현, no-op, 데모 데이터, 실제 SDK 교체 예정 지점은 반드시 주석으로 의도를 남긴다.
- 주석이 오래되어 거짓말을 하게 되는 경우가 가장 나쁘다. 코드 변경 시 관련 주석도 같이 갱신한다.

좋은 주석 예시:

```kotlin
// Pi가 재연결되면 이전에 못 보낸 pending 심박 샘플을 다시 밀어 넣습니다.
piNetworkDataSource.observeConnectionState().collect { state -> ... }
```

피해야 할 주석 예시:

```kotlin
// state를 collect한다.
piNetworkDataSource.observeConnectionState().collect { state -> ... }
```

## 문서화 원칙

- 사용자 흐름, 외부 연동, 프로토콜, 저장 구조가 바뀌면 `docs` 문서를 함께 갱신한다.
- README는 프로젝트 현재 상태와 실행/테스트 방법이 바뀔 때만 수정한다.
- 프로토콜 변경은 `docs/sleepcare_protocol_design.md` 또는 관련 계획 문서에 메시지 타입, 필드, 호환성 영향을 남긴다.
- Health Connect, Wear OS, Raspberry Pi 연동처럼 외부 시스템과 맞닿은 변경은 권한, 실패 상태, 재시도 흐름을 문서화한다.
- 문서는 길게 쓰기보다, 나중에 검증할 수 있는 사실과 결정 이유를 남긴다.

## 코드 변경 지침

- 기존 패턴을 먼저 따른다. 새 아키텍처나 큰 리팩터링은 필요할 때만 한다.
- `app`, `watch`, `watch-contracts` 사이의 계약을 바꿀 때는 양쪽 사용처와 테스트를 함께 확인한다.
- 공유 프로토콜은 문자열을 흩뿌리지 말고 `watch-contracts`의 경로/코덱/모델을 사용한다.
- DB Entity와 도메인 모델 변환은 `data/local` 경계에서 명확히 유지한다.
- ViewModel은 여러 Flow를 UI 상태로 합치는 역할에 집중하고, 복잡한 계산은 순수 함수나 Repository 쪽으로 분리한다.
- 사용자의 기존 변경은 되돌리지 않는다. 작업 범위 밖 변경은 그대로 둔다.

## 검증

코드 변경 후 가능한 경우 다음을 실행한다.

```powershell
.\gradlew.bat testDebugUnitTest
```

주석/문서만 바꿨더라도 Kotlin/Gradle 파일을 건드렸다면 테스트나 최소한 관련 컴파일 확인을 시도한다. Android SDK 접근이나 Build Tools 다운로드가 필요해 실패할 수 있으므로, 실패 이유가 환경 문제인지 코드 문제인지 결과에 명확히 남긴다.
