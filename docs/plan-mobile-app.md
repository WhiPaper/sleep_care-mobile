# Mobile App Plan

## 목표
- 전체 서비스의 중심 허브 역할을 수행한다.
- 기기 데이터를 수집, 저장, 분석, 시각화한다.
- 사용자 맞춤 수면 스케줄 제안을 제공한다.

## 주요 기능
- 라즈베리파이 디바이스 연결 및 상태 확인
- 스마트워치 데이터 연동
- 졸음 기록 타임라인 조회
- 수면 기록 조회
- 통합 분석 리포트
- 수면 스케줄 제안
- 학습 플랜 입력
- 시험 날짜 입력
- 알림 및 리마인더

## 기술 스택
### 앱 본체
- Kotlin
- Jetpack Compose + Material 3
- MVVM + Repository + UDF(단방향 데이터 흐름)
- Android 공식 가이드 기준으로 UI 레이어와 Data 레이어를 분리하는 방향을 따른다.

### 상태관리 / 비동기 / DI
- ViewModel
- Kotlin Coroutines + Flow
- Hilt
- BLE 수신 상태, 수면 기록 조회, 추천 결과 계산, 화면 상태 갱신을 분리하기 좋은 조합으로 본다.

### 로컬 저장
- Room: 졸음 이벤트, 수면 세션, 시험 일정, 학습 플랜, 추천 결과 캐시
- DataStore: 설정값, 마지막 동기화 시각, 연결된 기기 정보, 사용자 선호값

### 백그라운드 작업
- WorkManager
- 주기적 동기화, 리포트 계산, 재시도 작업에 사용한다.

### 라즈베리파이 통신
- BLE(GATT)
- 초기 페어링은 CompanionDeviceManager 우선 검토
- Android BLE 중앙 역할 기준으로 연결 흐름을 설계한다.

### 워치 수면 데이터
- 1차 선택: Health Connect

## 수면 스케줄 제안 기능
### 학습 플랜 기반
- 공부 시간표 또는 학습 플랜 입력
- 공부 시간대와 기상 시간을 반영한 취침 시간 제안
- 졸음이 잦은 시간대에 맞춘 휴식 가이드 제안

### 시험 날짜 기반
- 시험 날짜 입력
- 시험 시간에 맞춘 기상 리듬 조정 제안
- 시험 직전 기간의 취침 시간 가이드 제공

### 통합 추천 기준 초안
- 최근 수면 시간
- 수면 시각의 규칙성
- 공부 중 졸음 빈도
- 시험일까지 남은 기간
- 사용자가 원하는 기상 시간

## 화면 구조 초안
- 온보딩
- 기기 연결
- 홈 대시보드
- 졸음 분석
- 수면 분석
- 수면 스케줄 제안
- 학습 플랜 입력
- 시험 일정 관리
- 설정

## 구현 단계
1. 앱 골격 및 라우팅 구성
2. 라즈베리파이 데이터 수신 구현
3. 워치 데이터 수신 구현
4. 데이터 저장 구조 설계
5. 기초 리포트 및 시각화 구현
6. 규칙 기반 추천 MVP 구현
7. 학습 플랜 및 시험 일정 반영 고도화

## 기술 설계 메모
- 신규 화면은 Compose 기준으로 설계하고 Material 3 컴포넌트를 사용한다.
- 화면 상태는 ViewModel에서 관리하고 Flow 기반으로 UI에 전달한다.
- Repository 계층에서 BLE, 로컬 DB, 워치 데이터 소스를 분리한다.
- Room은 구조화된 분석 데이터 저장소로 사용하고 DataStore는 경량 설정 저장에 한정한다.
- 앱 재시작 이후에도 이어져야 하는 작업은 WorkManager로 처리한다.

## 협업 논의 포인트
- 홈 화면 핵심 카드 우선순위
- 학습 플랜 입력 UX 형식
- 분석 결과 시각화 방식
- 알림과 리마인더 빈도
- GPT 의견: 홈 화면은 "어제 수면 상태", "최근 졸음 빈도", "오늘 추천 취침 시간" 중심 구성이 적합하다.
- GPT 의견: 앱은 많은 정보를 보여주는 것보다 행동으로 이어지는 제안을 중심으로 설계하는 편이 좋다.
