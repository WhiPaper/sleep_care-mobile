## 워치 모듈 1차 구현 계획

### Summary
- 이번 구현은 `Galaxy Watch + Samsung Health Sensor SDK` 기반의 **별도 `:watch` 모듈**을 추가하고, 모바일 `:app`은 companion 앱으로 유지한다.
- 1차 범위는 **실시간 공부 세션 + 전체 워치 핸드셰이크 + 모바일 `hr.ingest` 중계 + 수면 데이터 연동**까지 포함한다.
- 수면 데이터는 워치에서 직접 읽지 않고, **폰에서 Health Connect**로 읽어 모바일 저장소와 추천 로직에 반영한다.
- 워치의 심박 세션은 **foreground tracking service**로 유지해서 화면이 꺼지거나 워치 UI를 벗어나도 계속 동작하게 한다.

### Implementation Changes
- 모듈 구조
  - `:watch`를 새 Wear OS 앱 모듈로 추가한다.
  - 워치/폰이 같이 쓰는 순수 계약은 새 `:watch-contracts` 공유 모듈로 분리한다.
  - `:watch-contracts`에는 `WatchFlushPolicy`, `WatchSessionConfig`, `WatchHeartRateSample`, `WatchHeartRateBatch`, `WatchCursor`, `WatchPaths`, `WatchProtocolCodec`, `WatchCursorCalculator`를 둔다.
  - `:app`에는 기존 Android 구현을 유지하고, Room 큐/커서, Pi 중계, 추천, Health Connect 읽기 로직을 둔다.

- 워치 앱 (`:watch`)
  - Compose for Wear OS로 화면을 구현한다.
  - 메인 흐름은 `Connection Waiting -> Active Session -> Alerting`, `Watch Settings`는 보조 진입 화면으로 둔다.
  - `Connection Waiting`, `Active Session`, `Alerting`은 1차부터 실제 상태와 연결한다.
  - `Watch Settings`는 1차에서 권한/연결/배터리/수면 연동 상태를 보여주는 반기능 화면으로 구현한다.
  - `MainActivity`는 워치 UI와 상태 표시를 담당한다.
  - `WearableListenerService`는 Data Layer 수신을 담당한다.
  - 별도 foreground tracking service가 `Samsung Health Sensor SDK` 연속 심박/IBI 수집을 담당하고, 샘플 버퍼와 세션 수명을 관리한다.
  - 워치는 최근 10분 심박 샘플을 메모리 또는 로컬 경량 저장소에 보관하고, reconnect 시 백필 응답이 가능해야 한다.

- 워치-폰 프로토콜
  - 기존 경로 외에 `session.ready`, `session.error`, `session.closed`를 1차부터 정의하고 구현한다.
  - 세션 시작 흐름은 `phone start -> watch prepare -> watch session.ready -> phone session.open -> pi session.ack`로 고정한다.
  - `session.error`는 최소 `code`, `message`, `recoverable`을 포함한다.
  - `session.closed`는 워치 tracking service와 listener 정리가 끝난 뒤 보낸다.
  - 런타임 데이터는 `hr.live` 또는 `hr.batch`, 폰 응답은 `hr.ack`, 누락 복구는 `backfill_req`, 경고는 `alert.vibrate`, 위험도 기반 조절은 `flush_policy`로 유지한다.
  - 폰은 `highest contiguous ack` 기준을 유지하고, gap 발견 시 반드시 `backfill_req(from=missing_seq)`를 보낸다.

- 모바일 앱 (`:app`)
  - 현재 추가된 워치 브리지/큐/커서를 `:watch-contracts` 기준으로 정리한다.
  - `StudySessionRepository`는 워치 `session.ready/error/closed`를 반영해 `ArmingWatch -> OpeningSession -> Running` 전이를 확정한다.
  - Pi 세션은 워치 준비 완료 전에는 열지 않는다.
  - `GalaxyWatchSessionDataSource` 역할은 폰 companion 쪽 Data Layer 브리지로 유지하되, 새 계약 타입으로 맞춘다.
  - Pi 중계는 수신 즉시 `hr.ingest`를 보내고, 실패 샘플은 기존 `watch_hr_samples` 큐에서 reconnect 시 재전송한다.

- 수면 데이터 연동
  - 수면 데이터는 폰에서 `Health Connect`로 읽는다.
  - 모바일 저장소의 `SleepRepository`를 실제 Health Connect 기반으로 교체한다.
  - 읽는 데이터는 최소 `SleepSessionRecord`와 가능한 경우 sleep stages까지 반영한다.
  - 추천 엔진과 분석 화면은 더 이상 “워치 준비 중”이 아니라 “Health Connect 미연결/권한 없음/데이터 없음” 상태를 구분해 표시한다.
  - 워치 앱은 수면 이력 자체를 조회하지 않고, `Watch Settings`에서 “수면 동기화는 휴대폰에서 관리됨” 상태만 보여준다.

### Public APIs / Interfaces / Types
- 새 모듈
  - `:watch`
  - `:watch-contracts`
- 새/변경 계약
  - `session.ready`
  - `session.error`
  - `session.closed`
  - 공유 `WatchProtocolCodec`와 `WatchPaths`
- 모바일 인터페이스
  - `WatchSessionDataSource`는 `start/stop`, `acknowledgeCursor`, `requestBackfill`, `updateFlushPolicy`, `sendVibrationAlert` 외에 ready/error/closed 상태를 노출해야 한다.
  - `StudySessionPhase`는 `ArmingWatch`를 유지하고, watch handshake 실패를 명시적으로 처리한다.
- 수면 인터페이스
  - `SleepRepository.refreshFromSource()`는 Health Connect 실제 읽기로 바뀐다.
  - 수면 empty state는 `권한 없음`, `Health Connect 미설치/비활성`, `데이터 없음`을 구분한다.

### Test Plan
- 프로토콜/계약
  - `session.start`, `session.ready`, `session.error`, `session.closed`, `hr.batch`, `hr.ack`, `backfill_req` 직렬화/역직렬화 테스트
  - gap 있는 샘플 수신 시 `highest contiguous ack`와 `pendingBackfillFrom` 계산 테스트
- 모바일 세션
  - 워치 `ready` 전에는 Pi 세션이 열리지 않는지
  - 워치 `error` 시 세션이 `Error`로 끝나고 Pi를 열지 않는지
  - reconnect 후 queued `hr.ingest`가 재전송되는지
- 워치 런타임
  - foreground tracking service 시작/중지
  - 화면 종료 후에도 심박 수집 지속
  - listener reconnect 후 `hr.batch` 백필 응답
  - `alert.vibrate` 수신 시 Alerting 화면 또는 alert state 반영
- 수면 연동
  - Health Connect 권한 허용/거부/데이터 없음/데이터 존재 시 상태 분기
  - 수면 세션이 있을 때 분석/추천이 실제 값으로 갱신되는지
- UI
  - `Connection Waiting`, `Active Session`, `Alerting` 상태 전이
  - `Watch Settings`의 연결/권한/수면 동기화 상태 표시

### Assumptions / Defaults
- 워치 앱은 **companion-dependent**로 두고 standalone 앱으로 만들지 않는다.
- 1차에서 Hilt는 `:watch`에 바로 넣지 않아도 되며, 필요하면 단순 상태 보관 + service locator 수준으로 시작한다.
- 워치 버퍼는 10분, 폰 큐는 세션 전체 유지, Pi는 기존 재전송 정책을 따른다.
- 수면 연동의 공식 소스는 Health Connect로 고정하고, Samsung Health Sensor SDK는 실시간 심박/IBI 용도로만 사용한다.
- `Watch Settings`는 1차에서 완전한 환경설정보다 상태/권한/연동 확인 화면에 집중한다.
