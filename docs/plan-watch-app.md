# Watch App Plan

## 목표
- 수면 데이터를 안정적으로 수집한다.
- 스마트폰 앱으로 전달 가능한 최소 기능을 구현한다.

## 주요 기능
- 수면 데이터 수집
- 스마트폰 앱과 동기화
- 동기화 상태 표시
- 향후 확장을 고려한 센서 데이터 구조 설계

## 기술 방향
- 1차 수면 데이터 소스는 Health Connect를 우선 검토한다.
- 스마트폰 앱은 Health Connect 기반 수면 기록 조회를 기준으로 데이터 구조를 설계한다.
- 워치 앱 범위는 수면 데이터 안정 수집과 동기화 검증에 우선 집중한다.

## 현재 범위
- 수면 시작/종료 시각 확보
- 총 수면 시간 기록
- 스마트폰 앱 연동 테스트

## 후속 확장 가능성
- 수면 단계 정보 활용 검토
- 심박 등 추가 데이터 활용 검토
- 장기 컨디션 보조 지표 활용 검토

## 구현 단계
1. 대상 워치 플랫폼 확정
2. 수면 데이터 접근 가능 여부 확인
3. 수집 가능한 최소 데이터 정의
4. 스마트폰 앱 동기화 구현
5. 동기화 실패 대응 처리

## 고려 사항
- 워치 OS별 API 차이
- 제조사별 제약
- 백그라운드 동기화 제한
- 배터리 소모 최소화

## 협업 논의 포인트
- 1차 대상 워치 플랫폼 선정
- 수면 데이터 소스 방식 결정
- 동기화 주기와 실패 처리 기준
- GPT 의견: 워치 앱은 수면 데이터 안정 수집에만 우선 집중하는 편이 현실적이다.
- GPT 의견: 플랫폼 확정이 늦어지면 전체 일정 리스크가 커질 수 있다.
